## 1. DESIGN SYSTEM

### 1.1 Color Palette (From Logo)

| Token | Hex | Usage |
|---|---|---|
| `--color-primary` | `#5B2D8E` | Deep Purple — nav, footer, headings, primary buttons |
| `--color-secondary` | `#2B3FAE` | Royal Blue — borders, links, icon rings, badges |

| `--color-white` | `#FFFFFF` | Card backgrounds, nav text on dark |

### 1.2 Typography

| Role | Font | Weight / Size |
|---|---|---|
| Primary Heading H1 | Playfair Display | 700 / 3–3.5rem |
| Sub-Heading H2 | Playfair Display | 600 / 1.8–2.2rem |
| Section Title H3 | Inter | 600 / 1.3–1.5rem |
| Body Copy | Inter | 400 / 1rem (16px) |
| Arabic Text | Scheherazade New | 400 / 1.2–1.6rem |
| UI Labels / Buttons | Inter | 600 / 0.85rem uppercase |
| Quote / Hadith | Cormorant Garamond Italic | 400 / 1.2rem |

### 1.3 Design Principles

1. **Use the HTML template layout exactly** — preserve structural skeleton, map content into it
2. **No generic AI layouts** — avoid cookie-cutter outputs; honour the template's design choices
3. **Purple-first identity** — deep purple is the dominant brand colour, don't dilute it
4. **Blue as structure** — royal blue for borders, circular elements, icon rings, dividers
5. **Islamic geometric motifs** as decorative SVG patterns at low opacity — never clipart
6. **Arabic text as a design asset** — not an afterthought; displayed prominently in key sections
7. **Real community photography** — no generic stock mosque images

```tsx
// components/home/UPIQRCode.tsx
// Install: npm install qrcode.react

import QRCode from 'qrcode.react';

const UPI_ID = "manhajambiya@upi"; // ← Replace with actual UPI ID from client
const PAYEE_NAME = "Manhaj Al Ambiya";

const upiURL = `upi://pay?pa=${UPI_ID}&pn=${encodeURIComponent(PAYEE_NAME)}&cu=INR`;

export function UPIQRCode() {
  return (
    <QRCode
      value={upiURL}
      size={220}
      fgColor="#5B2D8E"    // Deep purple
      bgColor="#FFFFFF"
      level="H"            // High error correction
      includeMargin={true}
    />
  );
}

// For preset amount buttons, append &am=AMOUNT to upiURL
// e.g. upi://pay?pa=...&am=500
```

## 8. ACCESSIBILITY & PERFORMANCE

### WCAG 2.1 AA
- All images: descriptive `alt` attributes including Arabic text descriptions
- Colour contrast: minimum 4.5:1 body text · 3:1 large text (deep purple on white passes)
- All interactive elements keyboard-navigable with royal blue focus rings
- Form fields: `<label>` elements + accessible error messages
- Audio player: screen reader labels, keyboard controls
- QR section: text fallback — "Or use UPI ID: [id]"
- Semantic HTML: `<main>`, `<nav>`, `<section>`, `<article>`, `<aside>`

### Performance Targets

| Metric | Target |
|---|---|
| Lighthouse (mobile + desktop) | ≥ 90 |
| Largest Contentful Paint | < 2.5s |
| Cumulative Layout Shift | < 0.1 |
| Images | WebP + AVIF fallback via `next/image` |
| Fonts | Preloaded via `next/font`, zero layout shift |
| Bundle | Tree-shaking, per-route code splitting |
